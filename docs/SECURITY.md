# 安全模型

`host-mcp` 是面向单用户本机环境的 MCP 服务，支持 Termux、Linux 和 WSL。它不应直接暴露到公共网络，也不应在没有额外安全边界的情况下通过反向代理、隧道或端口转发开放。

## HTTP 安全

- 默认仅监听 `127.0.0.1`。
- 管理员可以显式配置某个 IPv4 地址或 `0.0.0.0`，以允许可信局域网访问；优先使用特定接口地址，避免不必要地监听全部接口。
- 每个请求都必须通过 Bearer Token 认证。
- Token 先转换为固定长度 SHA-256，再使用恒定时间比较。
- Authorization Header 必须唯一且严格符合 `Bearer <token>` 格式。
- 浏览器 Origin 使用精确白名单匹配。
- 保留官方 MCP SDK 的 localhost Host-header 防护。
- 请求体、请求时间和 MCP 会话空闲时间均有限制。
- Token 轮换采用原子写入；轮换后的后续请求必须使用新 Token。

Bearer Token 不应放入 URL、查询参数、日志、截图或公开配置。

非回环监听是管理员主动开启的暴露面。当前传输是明文 HTTP：Bearer Token 提供认证，但不提供机密性或完整性保护，能观察网络流量的一方可能获取 Token、请求和响应。局域网访问只适合可信或隔离网络，并应配合主机防火墙限制来源；不可信 Wi-Fi、跨网段或公网访问应使用 VPN、SSH 隧道或经过审查的 TLS 终止层。MCP SDK 的 localhost Host-header 防护只保护回环监听场景，不能替代局域网防火墙或网络访问控制。

## 文件系统边界

每个命名 Root 都使用独立 `os.Root`。客户端路径采用 `{root, path}`：

```json
{
  "root": "workspace",
  "path": "src/main.go"
}
```

Root 内操作会阻止：

- 绝对路径
- NUL 字符
- `..` 目录穿越
- 中间或最终符号链接逃逸
- 未配置 Root
- 未授权操作
- 跨 Root 重命名

Linux/WSL 的系统目录硬拒绝规则和 Termux prefix/shared-storage 只读规则，会在配置校验和打开 Root 时重复验证。

不要授权包含设备节点、不可信挂载点、敏感凭据或其他用户数据的目录。`os.Root` 只能限制路径逃逸，不能判断某个挂载内容是否适合交给 AI Agent。

## 文件变更

- 写入和删除默认没有授权。
- 已有文件覆盖需要预期 SHA-256。
- 普通文件删除需要预期 SHA-256。
- 不支持递归删除。
- 拒绝删除符号链接。
- 目录、文件、读取和搜索结果都有边界限制。

## 命令执行

`command_status`、`exec_run` 和 `shell_run` 始终发布到 MCP 工具列表。禁用状态下，执行工具只返回设置提示，不启动进程；权限只能由设备所有者通过本地 CLI 修改。

### 受控命令 `exec_run`

`exec_run` 由内部 Go 函数 `RunExec` 实现。`RunExec` 不是用户需要调用或配置的另一个工具。

规则安全条件：

- 固定 executable，客户端不能选择其他程序。
- 直接使用 `os/exec`，不经过 Shell。
- 拒绝常见 Shell 和解释器。
- 工作目录必须位于规则允许的命名 Root 和子路径中。
- 参数、环境、时长和输出受限。
- 审计记录规则名和结果，不记录命令输出或敏感参数内容。

被允许的程序仍拥有 `host-mcp` 进程本身的操作系统权限。因此只应授权行为狭窄且可信的程序。

### 可信 Shell `shell_run`

可信 Shell 默认关闭，并要求本地专用风险确认。它允许任意 Shell 文本，因此不能保留受控命令的安全边界：

- Root/CWD 只决定 Shell 的起始目录，不限制后续 `cd` 或绝对路径访问；
- 内置文件工具对 Termux prefix 和共享存储的只读策略不约束 Shell；
- Shell 可以联网、启动子进程，并访问运行账户本身可访问的其他资源；
- 它不是容器、虚拟机、SELinux、AppArmor 或 seccomp 沙箱；
- 若监听地址不是回环地址，任何持有 Bearer Token 的网络客户端都可能使用该 Shell。

审计日志不保存命令正文，只保存其 SHA-256 摘要、起始 Root 和执行结果。

## 配置、Token 与审计

- 配置目录权限：`0700`
- 配置文件权限：`0600`
- Token 文件权限：`0600`
- 配置和 Token 使用同目录临时文件、`fsync` 和原子 rename 更新。
- 审计日志采用受限大小的 JSONL 轮换。
- 审计日志不记录 Token、Authorization Header、文件内容、原始 MCP 请求体、命令输出或环境秘密。

## 信任边界

`host-mcp` 提供的是应用层授权边界，不是执行不可信代码的完整沙箱。若要执行第三方或潜在恶意程序，应另行使用容器、虚拟机、独立系统用户、SELinux、AppArmor、seccomp 或其他操作系统隔离机制。
